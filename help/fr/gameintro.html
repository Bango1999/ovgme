Introduction

OvGME vous permet de gérer des Mods pour plusieurs jeux, logiciels ou autre destination personnalisée à travers la même instance. Une entrée de configuration définie les paramètres pour un répertoire de travail. Une configuration fonctionne avec trois composants: Une destination racine, un dossier stock de Mods, et un dossier de sauvegarde.

Dossier de destination racine

Une destination racine (Root path), est l'endroit où OvGME va installer les Mods que vous activez. Ce peut être un dossier d'installation de jeu ou de logiciel, ou tout autre dossier où vous souhaitez appliquez des changements.

Dossier stock de Mods

Le dossier stock de Mods est celui où OvGME va chercher les Mods disponibles. C'est le dossier dans lequel vous devez entreposez vos Mods pour cette destination racine (i.e. jeu, logiciel, etc.)

Dossier de sauvegarde

Le dossier de sauvegarde est celui où OvGME conserve tous les fichiers originaux de la destination racine (i.e. jeu, logiciel, etc.) qui sont modifiés par les Mods installés. Par defaut, OvGME conserve les données de sauvegarde dans sa propre structure interne (ProgramData\OvGME), mais vous pouvez choisir un dossier de sauvegarde personnalisé pour chaque configuration.

Note: Chaque destination racine de configuration doit être unique, vous ne pouvez pas créer plusieurs configurations avec la même destination racine.
