Dépôts de Mods

OvGME vous permet de configurer des dépôts de Mod distants pour le téléchargement ou la mise à jour de Mods. Ce chapitre ne traite de la fonctionnalité que vu du coté client. Si vous comptez héberger et gérer un dépôt, veuillez consulter le chapitre Gestion de dépôt.

Les dépôts de Mod sont des serveur web distants qui fournissent une liste de Mods à télécharger ou à mettre à jour. Ces serveurs doivent être compatibles OvGME pour être reconnus durant la requête des dépôts.

Note: les dépôts sont fournis par des tiers-parties, OvGME ne fournit aucun dépôt par défaut (voir Gestion de dépôt).

Configurer les dépôts

Pour configurer les dépôts distants pour la configuration actuel, allez dans le menu "Mods" sélectionnez "Repositories" puis "Configure...". Cela ouvrira une nouvelle boite de dialogue pour gérer les URL de dépôts.
Note: La liste des dépôts n'est pas globale, chaque configuration a sa propre liste de dépôts. Si vous configurez les dépôts pour une config, cette liste n'est valide que pour la config actuellement sélectionné.
Ajouter ou supprimer une URL
Pour ajouter une nouvelle URL de dépôt dans la liste, remplissez le champ de texte URL puis cliquez sur le bouton "Add". L'URL doit être une URL valide vers un serveur HTTP. Voici quelques exemples de format d'URL valides:
"http://www.example.com/repos/"
"www.example.com:8080/repository/"
"http://192.168.0.1:80/mods/"
Pour supprimer une URL de dépôt de la liste, sélectionnez l'URL désiré dans la liste et cliquez sur le bouton "Remove".
Activer ou désactiver une URL
Les dépôts peuvent être activés ou désactivé pour la requête. Un dépôt désactivé ne sera pas interrogé durant l'interrogation des dépôts, mais restera disponible dans la liste pour un usage future.
Pour activer ou désactiver une URL de dépôt, cochez ou décochez simplement la checkbox correspondant à l'URL dans la liste.

Interrogation des dépôts

En interrogeant les dépôts, OvGME va interroger chaque URL de dépôt activé pour créer la liste des Mods disponibles pour le téléchargement ou la mise à jour. Pour interroger les dépôts, allez dans le menu "Mods", sélectionnez "Repositories" puis "Query...". Cela vous ouvre une nouvelle boite de dialogue.

OvGME va automatiquement démarrer le processus d'interrogation. Une fois l'interrogation terminée, et si elle réussie, des Mods devraient apparaître dans la liste. Chaque entrée est identifié par une une icone qui décrit son statu relativement a votre liste de Mods locaux pour cette configuration.
: Signifie que le Mod est absent de votre dossier stock de Mods local et peut être téléchargé.
: Signifie que le Mod est déjà présent dans votre dossier stock de Mods local et est a la même version.
: Signifie que le Mod est déjà présent dans votre dossier stock de Mods local mais a une version inférieur à celle disponible dans les dépôts (vous pouvez upgrader ce Mod).
: Signifie que le Mod est déjà présent dans votre dossier stock de Mods local mais a une version supérieur à celle disponible dans les dépôts (si vous téléchargez ce Mod, cela va downgrader votre Mod local).
Téléchargement des Mods
Pour télécharger des Mods depuis les dépôts, vous pouvez simplement double-cliquer sur le Mod désiré dans la liste. Vous pouvez également sélectionner plusieur Mods, pui cliquer sur le bouton "Download selected". Finalement, vous pouvez synchroniser tous vos Mods locaux en cliquant sur le bouton "Download all".

Le téléchargement des Mods a plusieurs statu selon l'étape d'installation.
Pending... : Signifie que le Mod est dans la liste de téléchargement et en attente de procédure.
Disabling old... : Signifie que le Mod local actuel est en processus de désinstallation. En effet, un Mod doit être désactivé avant d'être mis à jour.
Downloading... : Signife que le Mod est en cours de téléchargement.
Il y'a aussi plusieurs statu d'erreur possibles:
Host not found : Signifie que la téléchargement a échoué car le nom d'hote n'a put être résolu.
Connection error : Signifie que la téléchargement a échoué à cause d'une erreur de connexion au serveur.
HTTP encode error : Signifie que la téléchargement a échoué à cause d'un encodage de transfert HTTP non supporté (voir ci-dessous pour les détails).
HTTP error ### : Signifie que la téléchargement a échoué à cause d'une erreur HTTP (généralement 404: page introuvable).
Connection lost : Signifie que la téléchargement a échoué à cause d'une connexion perdue.
I/O Open error : Signifie que la téléchargement a échoué à cause d'une erreur de création de fichier pour écriture sur le disque local.
I/O Write error : Signifie que la téléchargement a échoué à cause d'une erreur d'écriture de donnée dans le fichier sur le disque local.
Note: OvGME ne prend pas en charge le transfert de donnée HTTP chunked pour le téléchargement de Mod-Archive. Ceci n'est théoriquement par un problème puisque ce mode de transfert n'est utilisé que pour fournir des pages dynamiques tel que PHP ou ASP. Un fichier téléchargeable est vraiment très rarement transmit via la méthode de transfert chunked par un serveur web, et ce n'est pas une bonne pratique.
